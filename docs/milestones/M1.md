## Follow-up: M1.5 — Kill Tesseract Fallback

**Branch:** milestone/M1.5-kill-tesseract-fallback

### Why

M1 could not delete `extraction.py`, `tables.py`, and `manual_extracter.py` because `ocr_engine.py` imported them for the Tesseract OCR fallback path. That fallback is never reached in production — GPT-4o extraction succeeds on all invoice types we encounter, and the PRD specifies a human-review queue for unreadable docs, not a Tesseract retry.

Removing the fallback also orphans `constants.py`, `converter.py`, `preproces.py`, and `run_ocr.py` (only imported by `ocr_engine.py` and by each other in the Tesseract chain).

### What changed

**`ocr_engine.py` — rewritten imports and `process_invoice()`**

- Removed imports: `shutil`, `json`, `cv2`, `numpy`, `constants`, `converter`, `preproces`, `run_ocr`, `extraction`, `tables`
- Kept imports: `os`, `Path`, `load_dotenv`, `OpenAIExtractor`, `InvoiceData`, `GoogleSheetsService`, `get_credentials_path`, `QuickBooksService` (optional), `GLClassifier` (optional)
- `process_invoice()`: removed Tesseract fallback try/except, image preprocessing, temporary file handling. Now calls `extractor.extract_from_pdf()` or `extractor.extract_from_image()` directly and raises on failure.
- Module-level service initialization (extractor, sheets, gl_classifier, qbo) unchanged.

**7 files deleted (total ~450 lines):**

| File | Lines | Role |
|---|---|---|
| `extraction.py` | 107 | Regex-based field extraction from Tesseract output |
| `tables.py` | 172 | OpenCV table detection + cell parsing |
| `manual_extracter.py` | 59 | Manual override extraction when regex fails |
| `constants.py` | 12 | Shared mutable state (filename, paths) for legacy pipeline |
| `converter.py` | ~30 | pdf2image wrapper for Tesseract path |
| `preproces.py` | ~40 | OpenCV image preprocessing (grayscale, threshold, denoise) |
| `run_ocr.py` | ~30 | Tesseract subprocess wrapper |
